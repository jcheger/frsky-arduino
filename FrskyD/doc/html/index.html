FrskyD Documentation

Arduino library for Frsky D protocol.

This library is not designed to decode data on the transmitter, but on the receiver side. OpenTX makes the rest of the job on the transmitter.

This development is completely independent of Frsky or OpenTX.

What to know about D protocol

  • use an inverted serial communication at 9600bds
  • compared to SP, the D protocol has no slowness issue (no time based polling) - then sensor can take as much time it needs to send data. Anyhow, don't send too much data without delay. It could saturate the receiver buffer and corrupt data.

Receiver behavior

The receiver only waits for data

Hub behavior

The hub is only compatible with the D protocol. It sends frames, based on this rule:

frame period sensors
1 200ms accelerometers, altitude, temperature, voltage, current, rpm
2 1s GPS (except date and time), fuel level
3 5s GPS date and time

Some sensors behavior

Some sensors, such as VFAS, FLVS-01 or FVAS-02 can send D protocol telemetry without the hub.

Allthough, those sensors do not use the concept of frames, and send periodically their data. Such sensors must be chained, in order to serialize data, else transmission will conflit between sensors.

Packet format

byte(s) description
1 0x5E - header
1 sensor ID
2 (~4) data (1)
1 0x5E - footer (can be considered as next data header as well)

(1) there are 2 exceptions that must be re-encoded from 1 to 2 bytes:

  • 0x5E (header / footer) -> 0x5D 0x3E
  • 0x5D (exception marker) -> 0x5D 0x3D

Connection draft

  • pull down resistor on RX line (100k)
  • pull down resistor on TX line (100k)
